3 ## Usage
4 SignalIntegrity comes with a library called _SignalIntegrity.Lib_. From within your scripts, you type:
5 
6  import SignalIntegrity.Lib as si
7 
8 Then, all of the elements from within SignalIntegrity are accessible using si as the prefix. For example, the single frequency s-parameters of a current amplifier are found by call ing the function:
9 
10  si.d.CurrentAmplifier()
11 
12 All of the namespaces for these packages are listed at [Packages](http://teledynelecroy.github.io/SignalIntegrity/Doc/xhtml/namespaces.xhtml).
13 
14 All of the classes available to you are located at [Class List](http://teledynelecroy.github.io/SignalIntegrity/Doc/xhtml/annotated.xhtml) along with the [Class Hierarchy](http://teledynelecroy.github.io/SignalIntegrity/Doc/xhtml/inherits.xhtml).

48 ## Building the Package Documentation
49 The package documentation is generated using a tool called [Doxygen](http://www.doxygen.nl/).
50 Unfortunately, Doxygen is a tool made for C++ and Java and is not perfect for Python. I chose it originally because it is capable of creating [LaTeX](https://www.latex-project.org/) documentation that was originally intended with my book - that idea was abandoned. But still, it produces a very good documentation system.
51 
52 When properly set up, it is invoked from the [Doc](https://github.com/TeledyneLeCroy/SignalIntegrity/tree/master/Doc) directory using:
53 
54  doxygen SignalIntegrityWindows
55 
56 
57 or:
58 
59 
60  doxygen SignalIntegrityLinux
61 
62 
63 depending on which platform you're on.
64 
65 This uses the configuration provided in the [SignalIntegrityWindows](https://github.com/TeledyneLeCroy/SignalIntegrity/tree/master/Doc/SignalIntegrityWindows) or [SignalIntegrityLinux](https://github.com/TeledyneLeCroy/SignalIntegrity/tree/master/Doc/SignalIntegrityLinux) configuration file and creates a directory called xhtml with the web documentation. Both need to use a hack of [doxypy.py](https://github.com/TeledyneLeCroy/SignalIntegrity/tree/master/Doc/doxypy.py) that strips the triple quoted header required at the top of each Python file prior to processing. This filter is specified in the [SignalIntegrityLinux](https://github.com/TeledyneLeCroy/SignalIntegrity/tree/master/Doc/SignalIntegrityLinux) as:
66 
67  # The INPUT_FILTER tag can be used to specify a program that doxygen should
68  # invoke to filter for each input file. Doxygen will invoke the filter program
69  # by executing (via popen()) the command:
70  #
71  # <filter> <input-file>
72  #
73  # where <filter> is the value of the INPUT_FILTER tag, and <input-file> is the
74  # name of an input file. Doxygen will then use the output that the filter
75  # program writes to standard output. If FILTER_PATTERNS is specified, this tag
76  # will be ignored.
77  #
78  # Note that the filter must not add or remove lines; it is applied before the
79  # code is scanned, but not when the output code is generated. If lines are added
80  # or removed, the anchors will not be placed correctly.
81  #
82  # Note that for custom extensions or not directly supported extensions you also
83  # need to set EXTENSION_MAPPING for the extension otherwise the files are not
84  # properly processed by doxygen.
85 
86  INPUT_FILTER = ./doxypy.py
87 
88 Usually, Doxygen can find the hacked doxypy.py in the /SignalIntegrity/Doc/ directory - if it can't you might need to specify the path to this file directly.
89 
90 In [SignalIntegrityWindows](https://github.com/TeledyneLeCroy/SignalIntegrity/tree/master/Doc/SignalIntegrityWindows) this needs to be specified as:
91 
92  INPUT_FILTER = doxypy.bat
93 
94 Which uses the local [doxypy.bat](https://github.com/TeledyneLeCroy/SignalIntegrity/tree/master/Doc/doxypy.bat), which simply calls the input filter file.
95 
96 In order to render any equations in the documentation, you must have [GhostScript](https://www.ghostscript.com/download/gsdnld.html) installed.
97 
98 Currently package documentation is modified only the branch gh-pages. In other words, you should have another repository checked out on the gh-pages branch, generate this documentation, copy it to the gh-pages repository directory and check it in on gh-pages.
