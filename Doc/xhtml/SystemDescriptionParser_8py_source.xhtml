69  def _ProcessLine(self,line,exclusionList):
70  """processes the lines of a netlist
71 
72  Lines that can be processed at this level are processed and lines that
73  are unknown are place in a list of unknown lines for upstream processing. This
74  enables derived classes to benefit from what this class knows how to process and
75  to simply add specific functionality. As a simple example, a derived simulator class
76  needs to add output probes, and this simple system description class knows nothing of
77  this.
78 
79  netlist lines that are handled at this level are:
80  - 'device' - addition of devices.
81  - 'connect' - handles device connections.
82  - 'port' - adds system ports for s-parameter calculation.
83 
84  @param exclusionList list of strings representing commands to exclude. These are either
85  commands that are removed from the functionality (i.e. it would not be write to add a system
86  port in a simulation), or commands that are withheld until further processing at a later time
87  to enforce the order that commands are handled in. For example, we like the netlist to be agnostic
88  about the order of devices listed and device connections, but we cannot connect device ports until
89  a device has been declared.
90  """
