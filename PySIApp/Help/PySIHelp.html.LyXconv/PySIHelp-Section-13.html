Netlist

Netlists are the internal text based format of the schematic. There is a one-to-one correspondance between the netlist and the schematic from a device and connectivity standpoint.
each line in the netlist starts with one of the following keywords:
  • device↓- specifies one of the Built-in Devices (Parts)↑ in the schematic.
  • connect↓- specifies one or more device pin connections.
  • port↓- specifies a Port↑ and it’s connection to a device pin. (used only for S-parameter Generation↑ and Deembedding↑ applications).
  • meas↓- specifies a Measure Probe↑ and it’s connection to a device pin. (used only for Virtual Probing↑ applications).
  • output↓- specifies an Output Probe↑ and it’s connection to a device pin. (used only for Virtual Probing↑ and Simulation↑ applications).
  • stim↓- specifies a Stim↑ and it’s associated device pin. (used only for Virtual Probing↑ applications).
  • stimdef↓- specifies the relationship between defining and dependent stims (see Stim↑). (used only for Virtual Probing↑ applications).
  • unknown↓- specifies an Unknown↑ device (used only for Deembedding↑ applications).
  • system↓- specifies a System↑ device (used only for Deembedding↑ applications).
