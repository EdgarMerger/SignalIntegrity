stimdef

A netlist line beginning with the keyword stimdef specifies the relationship between defining and dependent stims in the schematic (see Stim↑).
This should only appear in Virtual Probing↑ applications.
A stimdef statement will only appear if there is a defining stim in the schematic.
The syntax is the keyword stimdef followed by a python list-based matrix definition where the form:
[[d11, d12, …, d1C], [d21, d22, …, d2C], …, [dR1, dR2, …, dRC]]
which is equivalent to a R × C matrix D:
⎛⎜⎜⎜⎜⎜⎝ d11 d12 … d1C         d21 d22 … d2C         ⋮ ⋮ ⋱ ⋮         dR1 dR2 … dRC ⎞⎟⎟⎟⎟⎟⎠
The matrix is such that for a C element vector of dependent stims m (with the elements ⎛⎜⎜⎜⎜⎜⎝ m1   m2   ⋮   mC ⎞⎟⎟⎟⎟⎟⎠), as listed in the netlist for the dependent stims and an R element vector of defining stims d (see stim↑ for the netlist statement for the stim and Stim↑ for a description of dependent and defining stims), that the stimdef defines D such that:
d = D⋅m
When no stimdef statement appears, there are C unknowns (and there must be at least C Measure Probe↑s specified with lines that begin with meas↑ in the netlist). If a stimdef statement appears, there are R unknowns (and there must be at least R Measure Probe↑s specified with lines that begin with meas↑ in the netlist).
