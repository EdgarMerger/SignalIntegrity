Part I Getting Started

Introduction

PySIApp is an application for performing certain important applications and tasks encountered in signal integrity. These basic applications are:
  1. Generation of s-parameters of systems.
  2. Linear Simulation of systems.
  3. Virtual probing systems.
  4. Deembedding s-parameters in systems.
The basic philosophy behind this tool can be summarized as follows:
  • A high percentage of tasks required by signal integrity engineers that are usually performed using expensive, closed, black box systems can be performed using this free, open-source tool.
  • While software tools are powerful and really help the signal integrity engineer, there are certain common mistakes made due to both quality of data input to the tools, and lack of basic understanding of what the tool is doing. We address these two issues by supplying many capabilities for understanding the quality of your input data, usually in the form of s-parameter files, and providing complete explanation of what the tool is doing all the way from theory (in the book that accompanies this software) and in the form of the software itself which can be examined and even modified.

Navigation

When PySIApp opens, the user sees a dialog with a menu bar, a toolbar, a canvas, and a status line. The menu bar, toolbar buttons and various key bindings enable certain tasks to be executed. The menu bar is broken into sections where appropriate. These are:
  • File- tasks for dealing with the project file.
  • Edit- tasks for undoing and redoing tasks in the schematic.
  • Parts- tasks for dealing with schematic elements in the drawing.
  • Wires- tasks for dealing with wires connecting schematic elements.
  • View- tasks for controlling the view of the schematic.
  • Calculate- tasks for performing calculations on the schematic.
  • Help- tasks dealing with the help system.

File

File tasks are broken into two basic areas:
  • Opening, Saving, and Clearing the project.
  • Exporting netlists and pictures of the drawing.

Parts

Parts tasks are for adding all elements of the schematic (except the connection wires). All elements of the schematic are parts, but some parts are are added specially:
  • Add Part- Adds generic parts to the schematic.
The remaining add tasks are for adding special parts used for specific PySI applications:
  • Add Port(used for calculating s-parameters and deembedding)
  • Add Output Probe(used for simulating and virtual probing)
  • Add Measure Probe(used only for virtual probing)
  • Add Stim(used only for virtual probing)
  • Add Unknown(used only for deembedding)
  • Add System(used only for deembedding)
If a part is selected, the following tasks are also possible:
  • Delete Part
  • Edit Properties
  • Duplicate Part
  • Rotate Part

  • Flip Vertically

Part II PySI Applications

PySI has the ability to perform four basic tasks:
  1. Generate s-parameters of systems.
  2. Simulate systems.
  3. Virtual probe systems.
  4. Deembed s-parameters in systems.
A system is, of course, a schematic containing various elements,. For the purpose of this section, schematic elements are broken into several categories. These are:
  1. Ports.
  2. Devices.
  3. Sources.
  4. Measure Probes.
  5. Output Probes.
  6. Stims.

Devices

Devices are the things you are most familiar with. They include resistive and reactive elements, dependent sources, opens, shorts (ground), transformers, etc. The most general and important element is the file which is a generic box whose s-parameters can be read from a file.

Ports

Ports define connections to circuit for the purpose of s-parameter generation and de-embedding. They are numbered and determine the port numbering of any s-parameters generated. At least one port must be present to perform the task of s-parameter generation or deembedding.

Sources

Sources define stimulus to the circuit for the purpose of simulation. There are various types of voltage and current generators which can generate steps, pulses, sinewaves, etc. The most important sources supply a waveform specified in a waveform file. At least one source must be present to perform simulation.

Output Probes

Output probes define the probing point for waveforms output by the system during either simulation or virtual probing. The waverforms at output probe connection points are what get displayed on the simulator plot dialog. At least one output probe must be present to perform simulation.
